| Track | Folder | Purpose |

1. Read progress files: `dev/PROGRESS.md`, `learning/PROGRESS.md`

## Development Team Perspectives

When making decisions, consider these expert viewpoints:

| Expert | Focus | Key Question |
|--------|-------|--------------|
| **Alex** (Tech Lead) | Architecture, Astro, Vercel, code quality | Is this the simplest design? |

| **Marcus** (Data) | Extractor logic, JSON schemas, ETL narrative | Does output look production-grade? |
| **Jordan** (UX) | Tailwind, animations, Core Web Vitals, conversion | Does this feel fast and premium? |
| **Sam** (DevOps) | Vercel config, env vars, CI, deployment | Can we ship this safely? |
| **Riley** (Product) | Lead funnel, Cal.com + Tally flow, conversion | Does this move toward a booked call? |

## Decision Principles

1. **$0 launch** - Free tiers only until revenue justifies spend
2. **Simulate first** - Real AI/backend only when necessary
3. **Ship incrementally** - Working homepage beats perfect architecture
4. **Convert** - If it doesn't help book calls, deprioritize
