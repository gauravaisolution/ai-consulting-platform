# Learning Path: Zero → Advanced

> **Pace:** Move as fast as you can absorb. No week deadlines.  
> **Style:** Learn in parallel with dev — each module unlocks the next dev phase.  
> **Current level:** Starting from zero

## How This Path Works

```
FOUNDATION ──► FRONTEND ──► INTERACTIVITY ──► BACKEND ──► DATA ──► GENAI ──► DEPLOY
     │              │              │              │          │        │         │
  Module 00      Module 01-03    Module 03      Module 04-05  06      07       08
```

You do **not** need to finish all learning before building. Learn the module that matches what you're building **right now**.

## Level Definitions

| Level | You can... |
|-------|-----------|
| **Basic** | Follow instructions, read code, make small edits |
| **Intermediate** | Build features independently, debug common errors |
| **Advanced** | Design systems, make architecture trade-offs, teach others |

## Per-Session Learning Flow

1. Open the module `README.md`
2. Complete one lesson (not the whole module in one go)
3. Apply it immediately in `dev/` if there's a matching dev task
4. Mark lesson complete in `learning/PROGRESS.md`
5. Ask me to review or explain anything unclear

## Checkpoint: Am I Ready for the Next Module?

Each module README ends with a **Checkpoint** section. Pass it before moving on — or tell me you're a fast learner and want to skip ahead; I'll quiz you quickly.
