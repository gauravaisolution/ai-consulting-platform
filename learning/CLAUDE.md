## Rules

1. **Pass checkpoint before advancing** - No skipping modules
2. **Learn what you're building** - Prioritize modules that sync with current dev work
3. **Code stays separate** - Learning exercises here; platform code in `dev/`
4. **No time limits** - Move at your own pace
